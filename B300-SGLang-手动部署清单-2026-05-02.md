# B300 SGLang 手动部署清单

按之前 `10.255.254.10` 的成功路径整理，适合在另一台 Ubuntu 24 的 B300 服务器上手动部署。

## 0. 前提

- 系统：Ubuntu 24
- GPU：8 x B300
- 模型目录已准备好，例如：`/data/model/Qwen3-0.6B`
- 优先使用 **NVIDIA open kernel driver**，不要默认走 closed `-server` 栈

## 1. 安装驱动和运行时

### 如果你有本地 NVIDIA repo 包

示例：

```bash
dpkg -i /home/ubuntu/nvidia-driver-local-repo-ubuntu2404-580.159.03_1.0-1_amd64.deb
cp -f /var/nvidia-driver-local-repo-ubuntu2404-580.159.03/*.gpg /usr/share/keyrings/
apt-get update
apt-get install -y nvidia-driver-pinning-580.159.03 || true

apt-get install -y \
  nvidia-driver-580-open \
  nvidia-fabricmanager \
  nvidia-imex \
  libnvidia-nscq \
  libnvsdm \
  nvlsm \
  infiniband-diags
```

### 关键注意

- 用本地 repo 时，优先按上面这组包名装
- 不要混装 Ubuntu 仓库里的 `nvidia-fabricmanager-580` / `nvidia-imex-580`，避免拖入不匹配依赖

## 2. 重启后验证主机 GPU 环境

```bash
modinfo -F license nvidia
nvidia-smi
systemctl status nvidia-fabricmanager --no-pager
```

### 期望结果

- `modinfo -F license nvidia` 输出：`Dual MIT/GPL`
- `nvidia-smi` 能看到 8 张 B300
- `nvidia-fabricmanager` 是 active

## 3. 修 Docker 的 GPU 运行时

如果 `docker --gpus all` 报这种错：

```text
docker: Error response from daemon: could not select device driver "" with capabilities: [[gpu]]
```

执行：

```bash
apt-get update
apt-get install -y nvidia-container-toolkit nvidia-container-toolkit-base
nvidia-ctk runtime configure --runtime=docker
systemctl restart docker
```

## 4. 先做 Docker GPU 冒烟测试

```bash
docker run --rm --runtime=nvidia --gpus all --ipc=host --entrypoint python3 \
  lmsysorg/sglang:v0.5.10.post1-cu130 \
  -c 'import torch; print(torch.cuda.device_count()); print(torch.cuda.get_device_name(0))'
```

### 期望结果

- 打印 `8`
- 打印 `NVIDIA B300 SXM6 AC`

## 5. 先用小模型拉起 SGLang

```bash
docker rm -f sglang-qwen3-0_6b || true
docker run -d \
  --name sglang-qwen3-0_6b \
  --restart unless-stopped \
  --runtime=nvidia \
  --gpus all \
  --ipc=host \
  --cap-add SYS_NICE \
  --ulimit memlock=-1 \
  --ulimit stack=67108864 \
  -p 30000:30000 \
  -v /data/model:/data/model \
  lmsysorg/sglang:v0.5.10.post1-cu130 \
  python3 -m sglang.launch_server \
  --model-path /data/model/Qwen3-0.6B \
  --host 0.0.0.0 \
  --port 30000 \
  --disable-cuda-graph
```

## 6. 验证 API 是否真的可用

### 模型列表

```bash
curl -fsS http://127.0.0.1:30000/v1/models
```

### 生成测试

```bash
curl -fsS http://127.0.0.1:30000/generate \
  -H 'Content-Type: application/json' \
  -d '{"text":"Hello from B300","sampling_params":{"max_new_tokens":16}}'
```

### 看容器状态和日志

```bash
docker ps --filter name=sglang
docker logs --tail 200 sglang-qwen3-0_6b
```

## 7. 大模型 / 8 卡模式

如果小模型验证通过，再换正式模型，常见改法是加 `--tp-size 8`：

```bash
docker rm -f sglang-main || true
docker run -d \
  --name sglang-main \
  --restart unless-stopped \
  --runtime=nvidia \
  --gpus all \
  --ipc=host \
  --cap-add SYS_NICE \
  --ulimit memlock=-1 \
  --ulimit stack=67108864 \
  -p 30000:30000 \
  -v /data/model:/data/model \
  lmsysorg/sglang:v0.5.10.post1-cu130 \
  python3 -m sglang.launch_server \
  --model-path /data/model/<model-dir> \
  --tp-size 8 \
  --host 0.0.0.0 \
  --port 30000 \
  --disable-cuda-graph
```

## 8. 常见坑

- 主机 `nvidia-smi` 正常，不代表 Docker 一定能用 GPU
- 容器是 `Up`，不代表 `/v1/models` 已经 ready
- Docker 化部署不要求主机必须有 `nvcc`
- 如果后面遇到 Triton / PTXAS / `sm_103a` 相关报错，优先怀疑镜像工具链，不要先怪主机驱动

## 9. 最小成功标准

满足下面 4 条，基本就算成了：

- `modinfo -F license nvidia` = `Dual MIT/GPL`
- `nvidia-smi` 看到 8 张 B300
- Docker 冒烟测试能看到 8 张 GPU
- `/v1/models` 和 `/generate` 都能返回正常结果
